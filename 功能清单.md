# CoverageTool 功能清单

生成时间：2026-06-15

## 1. 项目定位

当前仓库已经清理为通用 Windows 桌面端 OpenCppCoverage GUI。软件保留 Qt Widgets UI，不包含公司内项目识别、路径映射、缺陷系统集成、持续集成平台集成、FTP、在线更新、验证码、加密库拉取、QA 模式、特定项目过滤等内部功能。

核心目标是让使用者显式配置本地路径，然后启动 OpenCppCoverage 生成覆盖率报告或合并已有 `.cov` 文件。

## 2. 当前文件组成

仓库有效文件范围较小，主要由以下部分组成：

- `CMakeLists.txt`：当前源码构建入口，负责 Qt、Windows 资源和 qrc。
- `CoverageTool/main.cpp`：应用入口，初始化 Qt 应用、字体、图标和主窗口。
- `CoverageTool/CoverageTool.h`、`CoverageTool/CoverageTool.cpp`：全部业务 UI、设置、参数生成和进程执行逻辑。
- `CoverageTool/CoverageTool.qrc`、`CoverageTool/CoverageResource.rc`、`CoverageTool/Icon.*`：应用图标资源。
- `README.md`、`LICENSE`、`THIRD_PARTY_NOTICES.md`：开源说明、MIT 许可证和第三方边界说明。
- `build.ps1`、`rebuild.bat`：CMake 构建入口。

## 3. 当前可用功能

### 3.1 通用覆盖率配置

主界面提供以下显式配置项：

- OpenCppCoverage 环境检测状态。
- 目标可执行文件路径，并自动使用目标程序所在目录作为工作目录。
- 源码目录。
- PDB 源码路径替换源路径。
- 覆盖率输出目录。
- 排除源码路径列表。
- 额外 OpenCppCoverage 参数。
- 是否覆盖子进程。
- 是否启用 optimized build 支持。
- 界面语言：英文 / 简体中文。

这些字段全部由用户输入或选择，不再从注册表、项目映射表、内置项目列表或公司目录约定推导。

### 3.2 设置保存与加载

配置使用 `QSettings` 保存，当前仍存在的键为：

- `TargetExecutable`
- `SourceDirectory`
- `PdbSourcePath`
- `OutputDirectory`
- `MergeDirectory`
- `ExcludedSources`
- `ExtraArguments`
- `CoverChildren`
- `OptimizedBuild`
- `Language`

旧内部配置项已经不再参与保存或读取，例如应用名、提测包路径、缺陷系统产品名、公司日志等级、日志清理间隔、特定项目排除开关等。

### 3.3 覆盖率执行

点击 `Run coverage` 后，程序会：

1. 从 `PATH` 检测 OpenCppCoverage 程序，并校验目标 exe、源码目录、输出目录等输入。
2. 保存当前设置。
3. 创建输出目录及 `html` 子目录。
4. 基于显式配置生成 OpenCppCoverage 参数。
5. 使用 `QProcess` 启动 OpenCppCoverage。
6. 将标准输出和错误输出追加到 UI 日志面板。
7. 执行结束后记录输出目录或错误信息。

当前参数生成规则：

- `--modules <target exe 文件名>`
- `--sources <源码目录>`
- 可选 `--substitute_pdb_source_path <PDB源路径?源码目录>`
- 多个 `--excluded_sources <排除路径>`
- `--export_type=html:<输出目录>/html`
- `--export_type=binary:<输出目录>/coverage.cov`
- `--export_type=cobertura:<输出目录>/coverage.xml`
- `--verbose`
- 可选 `--optimized_build`
- 可选 `--cover_children`
- 追加用户填写的额外 OpenCppCoverage 参数
- `-- <目标 exe 完整路径>`

### 3.4 `.cov` 合并

点击 `Merge COV files` 后，程序会：

1. 校验 OpenCppCoverage 程序、待合并目录和输出目录。
2. 递归扫描待合并目录中的 `.cov` 文件。
3. 在输出目录下创建 `merge/<时间戳>` 子目录。
4. 为每个 `.cov` 文件生成 `--input_coverage=<file>` 参数。
5. 输出合并后的 HTML、binary COV 和 Cobertura XML：
   - `<merge目录>/`
   - `<merge目录>/merged.cov`
   - `<merge目录>/merged.xml`

### 3.5 日志与反馈

UI 保留日志面板，用于展示：

- 设置保存/加载状态。
- 覆盖率执行命令预览。
- OpenCppCoverage 标准输出。
- OpenCppCoverage 错误输出。
- 进程退出码。
- 进程启动错误。

日志只作为本地执行反馈，不再包含 FTP 上传、公司日志等级授权、清理策略或远程更新逻辑。

### 3.6 输出目录快捷打开

主界面保留两个本地打开动作：

- 打开最近一次覆盖率输出目录。
- 打开最近一次合并输出目录。

实现使用 `QDesktopServices::openUrl` 打开本地目录。

## 4. 当前依赖边界

### 4.1 构建依赖

- Windows。
- Visual Studio 2022 / v143 toolset。
- CMake 3.16 或更高版本。
- Qt 5.12.12 x64 CMake 包，使用模块 `Core`、`Gui`、`Widgets`。

### 4.2 运行依赖

- OpenCppCoverage 需要用户单独安装，并将 `OpenCppCoverage.exe` 加入 `PATH`。
- OpenCppCoverage 不再随仓库分发。
- 7-Zip 不再随仓库分发。

### 4.3 第三方二进制

当前仓库不再保留第三方预编译库。此前保留的 `components.lib` 未被当前源码使用，已经从 CMake 链接项和仓库文件中移除。

## 5. 已删除或不再存在的功能

当前源码和工程中不再保留以下功能入口：

- QA 模式、QA 页面、研发/QA 切换标签。
- 缺陷系统查询、数据导出和启动脚本。
- 持续集成平台客户端和集成逻辑。
- FTP 上传、在线更新、离线/在线包模式。
- 凭据管理、凭据生成脚本。
- 一次性验证码和相关生成逻辑。
- 加密库头文件、库、预构建拉取脚本和链接项。
- curl 目录和 libcurl 链接。
- 特定项目相关排除逻辑。
- 特定产品详细日志模式授权逻辑。
- 项目与路径映射、固定盘符映射、注册表项目识别。
- `OpenCppCoverage/` 内置源码或安装包。
- `tool/` 下的 EXE、DLL、安装包、更新脚本。
- 半成品迁移文档、内部重构状态文档、旧功能报告。

## 6. 当前代码结构评估

当前实现已经从多业务模式收敛到单窗口工具，业务集中在 `CoverageTool` 类内：

- UI 创建：`CoverageTool::buildUi`
- 路径选择与环境检测：`refreshCoverageStatus`、`openCoverageDownloadPage`、`browseTargetExe`、`updateWorkingDirectoryPreview`、`browseSourceDirectory`、`browsePdbSourcePath`、`browseOutputDirectory`、`browseMergeDirectory`
- 输入校验：`validateCoverageInputs`、`validateMergeInputs`
- 参数生成：`buildCoverageArguments`、`buildMergeArguments`
- 执行入口：`runCoverage`、`runMerge`
- 输出读取：`readProcessOutput`
- 进程状态：`onProcessFinished`、`onProcessError`、`setRunning`
- 配置持久化：`saveSettings`、`loadSettings`
- 语言切换：`onLanguageChanged`、`setLanguage`、`retranslateUi`、`uiText`

对开源版本而言，这个结构已经足够直接。若后续继续调整，可优先考虑把参数生成和设置模型从 UI 类中拆出，方便单元测试，但这不是当前开源清理的阻塞项。

## 7. 残留风险与建议

- 当前仓库不再保留第三方预编译二进制，发布材料主要需要覆盖 Qt 和 OpenCppCoverage 的外部依赖说明。
- `PdbSourcePath` 字段当前语义是 OpenCppCoverage 的 `--substitute_pdb_source_path` 替换源路径，不是“搜索 PDB 文件目录”。README 或 UI 文案后续可继续细化，避免用户误解。
- 当前没有自动化单元测试，验证主要依赖 CMake Debug/Release 编译和残留关键词扫描。

## 8. 本轮梳理结论

当前项目功能边界已经收敛为：

1. 自动检测 OpenCppCoverage 并配置目标程序。
2. 生成 HTML、binary COV、Cobertura XML 覆盖率报告。
3. 合并已有 `.cov` 文件并输出三种报告格式。
4. 保存和恢复本地通用配置。
5. 展示本地执行日志并打开输出目录。
6. 在英文和简体中文界面之间运行时切换。

这已经符合“通用 OpenCppCoverage GUI”的开源定位。剩余重点不是继续删除业务功能，而是完善 UI 文案，并在发布前补一次干净环境构建和实际覆盖率运行验证。
